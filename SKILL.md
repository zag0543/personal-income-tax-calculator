## 背景（痛点+后果+价值公式）

**痛点**: 个税计算涉及7项专项附加扣除、累计预扣法、年终奖临界点等多重规则，个人自主核算极易出错。每年3-6月汇算期，大量纳税人因不了解扣除政策和计税规则，多缴税却不知退税、少缴税收到补税通知才发现问题。

**后果**: 
- 专项附加扣除填报错误或遗漏：多缴税几百到几千元不自知
- 年终奖计税方式选错：临界点附近多发1元可能多交几千元税
- 累计预扣预缴与实际应纳税额差异：汇算时面临大额补税措手不及
- 多单位收入合并计税计算错误：补税+滞纳金风险

**价值**: 
- ✅ 精准计算年度应纳税额，与单位代扣对比，清晰知道退/补税金额
- ✅ 智能识别年终奖临界点，给出最优发放建议，避免"多发少得"
- ✅ 九项扣除自动汇总，支持个人养老金等最新优惠政策
- ✅ 提供可视化报告，方便向客户展示专业税务规划方案

## 核心计算逻辑

### 核心计算公式

```
应纳税所得额 = 综合所得收入额 - 60000 - 专项扣除 - 专项附加扣除 - 其他扣除
应纳税额 = 应纳税所得额 × 适用税率 - 速算扣除数
```

### 累计预扣预缴法

```python
def calculate_monthly_tax(month, monthly_income_list, monthly_deductions):
    """
    累计预扣预缴：每月按累计收入和累计扣除计算
    """
    cumulative_income = sum(monthly_income_list[:month])
    cumulative_deduction_fee = 5000 * month
    cumulative_taxable = cumulative_income - cumulative_deduction_fee - sum(monthly_deductions[:month])
    return cumulative_taxable
```

### 全年一次性奖金计税

```python
def calculate_bonus_tax(bonus, method='separate'):
    if method == 'separate':
        # 单独计税：奖金÷12找税率，单独计算税额
        monthly_bonus = bonus / 12
        # 查找适用税率和速算扣除数
        rate, quick_deduction = get_bonus_tax_rate(monthly_bonus)
        bonus_tax = bonus * rate - quick_deduction
    else:
        # 并入综合所得：与工资薪金合并计税
        bonus_tax = 0
    return bonus_tax
```

## 常见变体

### 变体1：多单位收入（跳槽、兼职多份工资）

```
情况：年中跳槽或同时在多个单位取得工资
处理：
1. 分别统计各单位的收入和已扣税
2. 按时间顺序合并累计计算
3. 技能自动按月分段汇总
```

**JSON示例：**
```json
{
  "multi_unit_income": true,
  "units": [
    {"months": 6, "monthly_income": 25000, "withheld": 12000},
    {"months": 6, "monthly_income": 35000, "withheld": 25000}
  ],
  "bonus": 30000,
  "bonus_method": "separate"
}
```

### 变体2：年终奖单独计税 vs 并入综合所得

```
情况：年终奖既可以单独计税，也可以并入综合所得
处理：技能自动对比两种方式的税额，给出最优建议
原则：
- 年终奖较低时（<36,000）：单独计税通常更优
- 年终奖较高时：需根据工资薪金金额综合计算对比
```

### 变体3：非居民个人年度中间离职

```
情况：外籍人士或非居民个人年度中间离境
处理：
1. 不再享受专项附加扣除（仅基本减除费用60000/年）
2. 按实际在华月份计算
3. 技能自动识别身份类型并调整计算逻辑
```

## 注意事项（Gotchas）⚠️ 最重要！

### ⚠️ 常见错误 #1：年终奖用错税率表

**错误做法**：将年终奖直接套用工资薪金7级超额累进税率表

**正确做法**：年终奖单独计税使用月度税率表（将年终奖除以12后找税率）

| 年终奖金额 | ÷12后金额 | 单独计税税率 | 税额 |
|------------|-----------|--------------|------|
| 36,000 | 3,000 | 3% | 1,080 |
| 36,001 | 3,000.08 | **10%** | **3,600.1** |
| 144,000 | 12,000 | 10% | 14,190 |
| 144,001 | 12,000.08 | **20%** | **28,800.2** |

**💡 小技巧**：年终奖在临界点附近（如36000、144000、300000）时，技能会自动预警并给出避税建议

---

### ⚠️ 常见错误 #2：赡养老人扣除不区分独生/非独生

**错误做法**：非独生子女也填报每月3000元（独生子女标准）

**正确做法**：
- **独生子女**：每月3000元，全年36,000元
- **非独生子女**：与兄弟姐妹分摊，每月每人最多1500元，需签订分摊协议

**💡 小技巧**：建议由税前收入最高的子女全额扣除，可以节省更多税款

---

### ⚠️ 常见错误 #3：专项附加扣除未按月足额填报

**错误做法**：只在有工资的月份填报扣除，或漏报部分月份

**正确做法**：
- 扣除项目一旦采集，全年有效（即使某月收入为0）
- 应在每年12月确认次年扣除信息
- 漏报的可在次年汇算时补充填报

**💡 小技巧**：技能支持按月输入或按年均输入，自动换算年度总额

---

### ⚠️ 常见错误 #4：个人养老金未在缴存当年扣除

**错误做法**：缴存个人养老金后忘记在个税APP填报扣除

**正确做法**：
1. 个人养老金账户资金封闭运行，领取时才征税
2. 缴存时即可填报扣除，当年即可享受税收优惠
3. 每年12月31日前缴存的才能在当年扣除

**💡 小技巧**：收入越高，边际税率越高，节省税款越多。高收入人群（税率20%+）每年缴存1.2万可节省2400元以上

---

### ⚠️ 常见错误 #5：职业年金与基本养老保险混淆

**错误做法**：将单位缴纳的职业年金也算入扣除

**正确做法**：
- 只能扣除**个人缴纳部分**（通常为缴费基数的4%）
- 单位缴纳部分不计入个人所得税扣除
- 技能中 `monthly_annuity` 字段仅填写个人部分

**💡 小技巧**：职业年金月扣除上限 = 缴费基数 × 4% ≤ 400元（企业年金职业年金个人缴费部分税前扣除标准为本人缴费工资计税基数的4%）

---

### ⚠️ 常见错误 #6：混淆公积金扣除上限

**错误做法**：公积金按实际缴纳金额全部扣除

**正确做法**：
- 公积金个人缴纳部分可全额扣除（无明确上限）
- 但需确认缴费比例是否合规（5%-12%）
- 超过合规比例的部分需并入工资薪金计税

**💡 小技巧**：合规范围内公积金缴纳越高，可抵扣金额越多，但实得工资会相应减少

---

### ⚠️ 常见错误 #7：年终奖临界点"多发少得"

**错误现象**：多发1元年终奖，到手反而更少

| 实际发放 | 税额 | 税后收入 | 对比 |
|----------|------|----------|------|
| 36,000元 | 1,080元 | 34,920元 | 基准 |
| 36,001元 | 3,600元 | 32,401元 | **多发1元，少拿2,519元** |
| 144,000元 | 14,190元 | 129,810元 | 基准 |
| 144,001元 | 28,800元 | 115,201元 | **多发1元，少拿14,609元** |

**💡 小技巧**：年终奖临界点预警功能（v1.1.0+）会自动识别并提示，可参考建议调整发放金额
